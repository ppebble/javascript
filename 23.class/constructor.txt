constructor property

일반적으로 클래스를 만들면 자동으로 constructor라는 속성이 만들어진다.
이 속성은 해당 클래스의 생성자 정보가 기본값으로 담기게 되며 
상속관계를 맺는 경우 자식클래스의 constructor엔 기본적으로 부모클래스의 생성자가 담기게된다.
 = constructor로는 자식 객체의 클래스는 알아낼 수 없다.
   --명시하면 됨

   라이브러리를 만들 때 사용하게 되는 속성

클래스 상속 규칙 ?

코드 재사용만을 위해 클래스 상속을 사용하면 안된다
중복 코드 제거만을 위해 클래스 상속을 사용하면 안된다.
클래스 상속은 패밀리 관계(is a)를 유지하면서 기능을 확장할 때만 사용해야 한다.
